For this assignment, I used class examples, previous project code, and some outside help to understand how to build a particle system and manage game state. I focused on creating a Particle class and integrating it into gameplay so that particles appear when enemies are damaged or defeated.

I also expanded my project by adding multiple enemy types with different movement patterns, including horizontal and diagonal motion. I implemented a health system where enemies take damage when the player jumps on them, and are removed when their health reaches zero. I used arrays to manage enemies, particles, and food items, which helped keep the code organized and scalable.

One of the biggest challenges was making sure all systems worked together without breaking existing features, especially when adding levels, enemy spawning, and particle effects. Debugging interactions between player movement, collisions, and enemy behavior required careful testing.

A major improvement I made was creating a level system where the player must defeat a certain number of enemies and collect food to progress. This added structure and made the game feel more complete.

Overall, I learned how to extend a game in a controlled way, how to organize code across multiple files, and how to use visual feedback like particles to improve clarity and player experience.
